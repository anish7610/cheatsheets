Let's Encrypt Command Guide

Description

The letsencrypt command is a client for the Let's Encrypt Certificate Authority (CA). It automates the process of obtaining and renewing SSL/TLS certificates for your web server.

Common Options

Option Description
certonly Obtain or renew a certificate without installing it.
renew Renew all previously obtained certificates that are near expiry.
--apache Use the Apache plugin for certificate installation.
--nginx Use the Nginx plugin for certificate installation.
--webroot Use the webroot plugin for authentication.
--dry-run Perform a test run of the issuance process.
-d Specify the domain names for the certificate.

Usage Examples 

# Obtain a certificate for example.com using Apache
letsencrypt --apache -d example.com

# Obtain a certificate using webroot for two domains
letsencrypt certonly --webroot -w /var/www/html -d example.com -d www.example.com

# Renew all certificates
letsencrypt renew

# Test obtaining a certificate (without affecting existing certs)
letsencrypt certonly --webroot -w /var/www/html -d example.com --dry-run

Cheat Sheet 

# Apache installation
letsencrypt --apache -d domain.com

# Nginx installation
letsencrypt --nginx -d domain.com

# Webroot authentication
letsencrypt certonly --webroot -w /path/to/webroot -d domain.com

# Renew certificates
letsencrypt renew
