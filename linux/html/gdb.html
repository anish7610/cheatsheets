GDB Command Overview

Description

GDB, the GNU Debugger, allows you to see what is happening inside your program while it runs, or what the program was doing before it crashed. It is a powerful tool for debugging applications written in languages such as C and C++.

Common Options

Option Description
-q Run quietly, suppressing introductory messages.
-ex <command> Execute the specified command at startup.
-tui Enable the Text User Interface.
--args Pass arguments to the program being debugged.
-p <pid> Attach to a running process by its process ID.

Usage Examples 


# Start debugging a program
gdb ./program

# Run a program with arguments
gdb --args ./program arg1 arg2

# Run quietly and execute commands from a script
gdb -q -x commands.txt ./program

# Attach to a running process
gdb -p 12345

Cheat Sheet 


# Basic usage
gdb ./program

# Pass arguments to program
gdb --args ./program arg1 arg2

# Attach to process
gdb -p 

# Run TUI
gdb -tui ./program

# Suppress messages
gdb -q ./program
