systemd-nspawn Command

Description

systemd-nspawn is a tool to run a command or OS in a light-weight container. It's useful for testing and building self-contained environments similar to chroot but with more isolation and security features.

Common Options

Option Description
-D, --directory= Specify the directory to use as the container's root.
-b, --boot Boot up the container image.
--machine= Assign a hostname to the container.
--network-veth Enable a virtual Ethernet link between host and container.
-n, --private-network Disconnect networking of the container from the host.
--bind= Bind mount a file or directory from the host to the container.
--read-only Mount the container's root as read-only.

Usage Examples 


# Run a command in a light-weight container
systemd-nspawn -D /srv/mycontainer /bin/bash

# Boot a container
systemd-nspawn -bD /srv/mycontainer

# Assign a hostname
systemd-nspawn -D /srv/mycontainer --machine=mycontainer

# Create a container with a virtual Ethernet link
systemd-nspawn -D /srv/mycontainer --network-veth

# Run with read-only root
systemd-nspawn -D /srv/mycontainer --read-only

Cheat Sheet 


-D, --directory=     # Set container root directory
-b, --boot                # Boot container
--machine=          # Set container hostname
--network-veth            # Create veth link
-n, --private-network     # No host network
--bind=             # Bind mount host path
--read-only               # Read-only container root
