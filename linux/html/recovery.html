Linux Command: recovery

Description

The recovery command is used to attempt recovery of a corrupted file system. It's often employed when a file system becomes inconsistent, primarily dealing with ext-based file systems.

Common Options

Option Description
-f Force the recovery even if the file system appears to be clean.
-b blocknum Specify an alternative superblock, useful for scenarios where the primary superblock is corrupted.
-n Run the command in 'No-Write' mode for a safe dry-run.

Usage Examples

Basic Recovery 

sudo recovery /dev/sdX1

Using an Alternative Superblock 

sudo recovery -b 32768 /dev/sdX1

Force Recovery Regardless of State 

sudo recovery -f /dev/sdX1

Dry Run to Check for Issues 

sudo recovery -n /dev/sdX1

Cheat Sheet 


recovery -f /dev/sdX1   # Force recovery
recovery -b 32768 /dev/sdX1   # Use alternative superblock
recovery -n /dev/sdX1   # No-Write/Dry-run
