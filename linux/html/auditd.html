Linux auditd Command Overview

Description

The auditd command is the Linux Audit daemon responsible for logging events on a system. It captures audit events generated by the Linux kernel and other components, providing a comprehensive audit framework.

Common Options

Option Description
-f Run auditd in the foreground for debugging.
-n Disable daemon startup messages for quieter startup.
-s Stop the audit daemon and flush logs.
-c Cache size to hold audit rules before sending to kernel.
-r Specify the number of retry attempts if logs can't be written.

Usage Examples 

        # Start the audit daemon with default settings
        sudo service auditd start

        # Stop the audit daemon and flush logs
        sudo auditd -s

        # Run auditd in foreground for debugging purposes
        sudo auditd -f

        # Set cache size to 32 for audit rules
        sudo auditd -c 32

Cheat Sheet
