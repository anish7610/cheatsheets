systemd-analyze

Description

The systemd-analyze command is used to analyze and debug performance issues with system boot-up and service loading times.

Common Options

Option Description
blame Lists system services ordered by the time they take to initialize.
critical-chain Shows the critical chain of units and their timings, highlighting the main performance bottlenecks.
plot Generates an SVG graphic illustrating the timing of the system boot-up.
time Prints basic startup timing data, including the kernel, initrd, and userspace times.
verify Checks unit files for correctness.

Usage Examples 

# Analyze system boot-up time
systemd-analyze time

# List services by initialization time
systemd-analyze blame

# Display the critical chain of boot dependencies
systemd-analyze critical-chain

# Generate a graph of boot-up processes
systemd-analyze plot > boot-process.svg

# Verify unit files for errors
systemd-analyze verify my-service.service
