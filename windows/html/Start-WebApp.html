Start-WebApp Command Overview

The Start-WebApp command is used in Windows to start a stopped web application within Internet Information Services (IIS). This is useful when managing deployed web applications.

Common Options

Option Description
-Name Specifies the name of the web application to start.
-Site Indicates the parent website of the application.
-Confirm Prompts for confirmation before starting the application.
-WhatIf Shows what would happen if the command is run, without executing it.

Usage Examples 

// Start a web application named 'MyApp' under 'MySite'
Start-WebApp -Name 'MyApp' -Site 'MySite'

// Prompt for confirmation before starting
Start-WebApp -Name 'MyApp' -Site 'MySite' -Confirm

// Check what would occur without execution
Start-WebApp -Name 'MyApp' -Site 'MySite' -WhatIf

Quick Reference Cheat Sheet 

Start-WebApp -Name <AppName> -Site <SiteName>
-Confirm // Request confirmation
-WhatIf // Simulate the action
