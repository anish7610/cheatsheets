Get-NetRoute Command Overview

The Get-NetRoute command in Windows PowerShell retrieves the IP routing table, displaying IPv4 and IPv6 routes on the system. It's useful for examining network paths and diagnosing routing issues.

Common Options

Option Description
-InterfaceIndex Filters routes by the specified network interface index.
-DestinationPrefix Shows routes matching the specified destination prefix.
-NextHop Displays routes using the specified next hop IP address.
-AddressFamily Specifies address family: IPv4, IPv6.

Usage Examples 

# Retrieve all routes
Get-NetRoute

# Filter routes by interface index 3
Get-NetRoute -InterfaceIndex 3

# Display routes with a specific destination prefix
Get-NetRoute -DestinationPrefix "192.168.1.0/24"

# Show IPv6 routes only
Get-NetRoute -AddressFamily IPv6

# Find routes with a specific next hop
Get-NetRoute -NextHop "10.0.0.1"

Quick Reference Cheat Sheet 

Get-NetRoute                      # All routes
Get-NetRoute -InterfaceIndex n   # Routes by interface
Get-NetRoute -DestinationPrefix subnet # Routes by subnet
Get-NetRoute -NextHop IP         # Routes by next hop
Get-NetRoute -AddressFamily IPv4|IPv6   # Routes by address family
