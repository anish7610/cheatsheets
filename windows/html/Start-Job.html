Start-Job Command Overview

The Start-Job command in Windows PowerShell is used to run commands or scripts asynchronously in the background. This allows users to perform other tasks in the console while the job executes.

Common Options

Option Description
-ScriptBlock Specifies the script block to run as a background job.
-Name Assigns a name to the job.
-ArgumentList Provides a list of arguments for the script block.
-Credential Runs the job using specified user credentials.
-InitializationScript Runs a script block on the remote session before the main script block.

Usage Examples

Example 1: Simple Background Job 

Start-Job -ScriptBlock { Get-Process }

Example 2: Job with a Name 

Start-Job -ScriptBlock { Get-EventLog -LogName Security } -Name "SecurityLogJob"

Example 3: Passing Arguments 

Start-Job -ScriptBlock { param($name) Get-Process -Name $name } -ArgumentList "notepad"

Example 4: Using Credential 

$cred = Get-Credential
Start-Job -ScriptBlock { Get-Service } -Credential $cred

Cheat Sheet 


# Start a simple background job
Start-Job -ScriptBlock {  }

# Name a job
Start-Job -ScriptBlock {  } -Name "JobName"

# Run with arguments
Start-Job -ScriptBlock { param($args)  $args } -ArgumentList 

# Use credentials
$cred = Get-Credential
Start-Job -ScriptBlock {  } -Credential $cred
