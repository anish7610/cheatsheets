New-ScheduledTask Command

Description

The New-ScheduledTask command in PowerShell creates a scheduled task definition object, which can be used to register new tasks with Register-ScheduledTask . It is primarily used to automate repetitive tasks on Windows systems.

Common Options

Option Description
-Action Specifies the action (script or program) to execute when the task runs.
-Trigger Defines the conditions that trigger the task, such as a daily schedule.
-Principal Sets the user account context in which the task runs.
-Settings Specifies additional settings for the task, like waking the computer to run the task.
-Description Adds a description to the scheduled task.

Examples

Daily Backup Task 


$action = New-ScheduledTaskAction -Execute "C:\Scripts\backup.ps1"
$trigger = New-ScheduledTaskTrigger -Daily -At 3AM
$principal = New-ScheduledTaskPrincipal -UserId "DOMAIN\User" -LogonType Password
$task = New-ScheduledTask -Action $action -Trigger $trigger -Principal $principal
Register-ScheduledTask -TaskName "DailyBackup" -InputObject $task

Weekly Log Cleanup 


$action = New-ScheduledTaskAction -Execute "C:\Scripts\cleanup_logs.ps1"
$trigger = New-ScheduledTaskTrigger -Weekly -DaysOfWeek Sunday -At 2AM
$settings = New-ScheduledTaskSettingsSet -AllowStartIfOnBatteries
$task = New-ScheduledTask -Action $action -Trigger $trigger -Settings $settings
Register-ScheduledTask -TaskName "WeeklyLogCleanup" -InputObject $task

Cheat Sheet 


Action    => New-ScheduledTaskAction -Execute "command"
Trigger   => New-ScheduledTaskTrigger -Daily|-Weekly -At time
Principal => New-ScheduledTaskPrincipal -UserId "user" -LogonType type
Settings  => New-ScheduledTaskSettingsSet -Option
Task      => New-ScheduledTask -Action act -Trigger trig -Principal prin -Settings set
Register  => Register-ScheduledTask -TaskName "name" -InputObject task
