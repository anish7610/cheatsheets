Get-EventLog Command Overview

Description

The Get-EventLog command in Windows PowerShell retrieves the events from Windows event logs, useful for system monitoring and troubleshooting.

Common Options

Option Description
-LogName Specifies the name of the event log (e.g., Application, System).
-Newest Returns a specified number of the most recent events.
-EntryType Filters events by type (e.g., Error, Information, Warning).
-Source Filters events by source name.
-After Returns events that occur after a specified date and time.
-Before Returns events that occur before a specified date and time.

Usage Examples 

# Retrieve the last 5 events from the System log
Get-EventLog -LogName System -Newest 5

# Retrieve all error events from the Application log
Get-EventLog -LogName Application -EntryType Error

# Get events from a specific source within the System log
Get-EventLog -LogName System -Source "Service Control Manager"

# Find events after a specific date in the Application log
Get-EventLog -LogName Application -After (Get-Date).AddDays(-1)

Cheat Sheet 

# Get recent events
Get-EventLog -LogName [LogName] -Newest [N]

# Filter by type
Get-EventLog -LogName [LogName] -EntryType [Type]

# Filter by source
Get-EventLog -LogName [LogName] -Source [SourceName]

# Date range filter
Get-EventLog -LogName [LogName] -After [Date] -Before [Date]
