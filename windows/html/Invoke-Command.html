Invoke-Command Overview

The Invoke-Command cmdlet in Windows PowerShell is used to run a command or script on a local or remote computer, allowing for automation across multiple systems.

Common Options

Option Description
-ScriptBlock Specifies the command or script to run.
-ComputerName Defines the target computer(s) to run the command on.
-Credential Supplies user credentials for authentication.
-FilePath Points to a script file to be executed.
-ArgumentList Provides arguments for the script or command.

Usage Examples 

# Run a script block on a remote computer
Invoke-Command -ScriptBlock { Get-Process } -ComputerName "Server01"

# Execute a script file on multiple computers
Invoke-Command -FilePath "C:\Scripts\Update.ps1" -ComputerName "Server01", "Server02"

# Using credentials for authentication
Invoke-Command -ScriptBlock { Get-Service } -ComputerName "Server01" -Credential (Get-Credential)

Cheat Sheet 


# Execute a command locally
Invoke-Command -ScriptBlock {  }

# Execute a command on a remote computer
Invoke-Command -ScriptBlock {  } -ComputerName "RemotePC"

# Run a script with arguments
Invoke-Command -FilePath "C:\MyScript.ps1" -ArgumentList 

# Use specific user credentials
Invoke-Command -ScriptBlock {  } -ComputerName "RemotePC" -Credential (Get-Credential)
