Register-ScheduledTask Command

Description

The Register-ScheduledTask command in Windows PowerShell is used to create and register a scheduled task, allowing automation of tasks on a set schedule or event.

Common Options

Option Description
-Action Specifies the action to execute. Use New-ScheduledTaskAction to define it.
-Trigger Specifies the task trigger. Use New-ScheduledTaskTrigger to define it.
-TaskPath Defines the folder path for the task in Task Scheduler.
-User Specifies the user account under which the task runs.
-Description Provides a description for the task.
-RunLevel Sets the privileges for the task ( Limited or Highest ).

Examples

Create a Daily Backup Task 

Register-ScheduledTask -Action (New-ScheduledTaskAction -Execute "backup.bat") -Trigger (New-ScheduledTaskTrigger -Daily -At "3AM") -TaskPath "\Maintenance" -User "SYSTEM" -Description "Daily backup task"

Run a Script at User Login 

Register-ScheduledTask -Action (New-ScheduledTaskAction -Execute "C:\Scripts\loginscript.ps1") -Trigger (New-ScheduledTaskTrigger -AtLogOn) -User "DOMAIN\User" -Description "Run script at user login"

Cheat Sheet 


Register-ScheduledTask -Action  -Trigger  [-TaskPath ] [-User ] [-Description ] [-RunLevel ]
