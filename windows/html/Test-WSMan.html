Test-WSMan Command Overview

The Test-WSMan command is used to verify the availability and accessibility of the Windows Remote Management (WinRM) service on a local or remote machine.

Common Options

Option Description
-ComputerName Specifies the remote machine to test. Default is the local computer.
-Credential Specifies a user account that has permission to perform this action on the remote computer.
-Port Specifies the port number on the remote computer.
-UseSSL Indicates that the connection should use SSL. Default is HTTP.

Usage Examples 

# Test the local machine
Test-WSMan

# Test a remote machine by name
Test-WSMan -ComputerName "server01"

# Test a remote machine with specific credentials
Test-WSMan -ComputerName "server01" -Credential (Get-Credential)

# Test a remote machine on a specific port
Test-WSMan -ComputerName "server01" -Port 5986

# Test with SSL
Test-WSMan -ComputerName "server01" -UseSSL

Cheat Sheet 

# Local test
Test-WSMan

# Remote test
Test-WSMan -ComputerName "hostname"

# With credentials & SSL
Test-WSMan -ComputerName "hostname" -Credential (Get-Credential) -UseSSL
