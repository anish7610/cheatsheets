xcopy Command Overview

The xcopy command in Windows is used to copy files and directories, including their subdirectories. It provides more options than the standard copy command, making it suitable for duplicating complex directory structures.

Common Options

Option Description
/s Copy directories and subdirectories except empty ones.
/e Copy all subdirectories, including empty ones.
/v Verify each file as it is written to the destination.
/f Display full source and destination file names while copying.
/h Copy hidden and system files as well.
/k Copy attributes. Normal xcopy will reset read-only attributes.
/d[:MM-DD-YYYY] Copy files changed on or after the specified date. If no date is provided, copies only if source is newer than destination.
/i If destination does not exist, assume it's a directory.
/q Do not display file names while copying.

Usage Examples 

xcopy C:\SourceFolder D:\Backup /s /e /v /f

Copy all files and subdirectories from C:\SourceFolder to D:\Backup , including empty subdirectories, with verification and full path display. 

xcopy C:\SourceFolder D:\Backup /d /e /h /i

Copy only files that are newer from C:\SourceFolder to D:\Backup , including hidden files and assuming the destination is a directory.

Cheat Sheet 

xcopy [source] [destination] [options]
  /s    # Copy subdirectories
  /e    # Include empty directories
  /v    # Verify files after writing
  /f    # Show full file paths
  /h    # Include hidden/system files
  /d    # Only newer files
  /i    # Treat destination as folder
