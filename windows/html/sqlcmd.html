sqlcmd Command

Description

The sqlcmd utility is a command-line tool in Windows used to execute SQL queries and manage SQL Server instances. It allows users to connect to SQL Server, run T-SQL commands, and perform database operations directly from the terminal.

Common Options

Option Description
-S Specifies the SQL Server instance name or IP address.
-U Specifies the user login ID for SQL Server authentication.
-P Specifies the user password for SQL Server authentication.
-Q Executes a T-SQL query provided as a string.
-i Specifies the file path of a script to be executed.
-o Specifies the output file for results.
-E Uses integrated Windows authentication.
-d Specifies the initial database to connect to.

Usage Examples 

# Connect to SQL Server using Windows authentication and execute a query
sqlcmd -S localhost -E -Q "SELECT * FROM my_table"

# Connect to SQL Server using username and password, and execute a script file
sqlcmd -S localhost -U my_user -P my_pass -i script.sql

# Execute a query and save the output to a file
sqlcmd -S localhost -E -Q "SELECT * FROM my_table" -o output.txt

Cheat Sheet 

# Connect with Windows auth and run a query
sqlcmd -S [server] -E -Q "[query]"

# Connect with SQL auth
sqlcmd -S [server] -U [user] -P [password] -Q "[query]"

# Run script file
sqlcmd -S [server] -U [user] -P [password] -i [file.sql]
