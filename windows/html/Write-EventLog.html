Write-EventLog Command

The Write-EventLog command in Windows PowerShell is used to write an event to a specified event log. It's useful for custom logging and tracking specific events or errors.

Common Options

Option Description
-LogName Specifies the name of the event log. Example: Application, System.
-Source Indicates the source of the event. Must be registered before it's used.
-EventID Specifies the Event ID for the log entry. Useful for identification.
-EntryType Specifies the type of event (e.g., Error, Information, Warning).
-Message The message to be written to the event log. Describes the event.

Usage Examples 

# Writing an informational event
Write-EventLog -LogName Application -Source MyApp -EventID 101 -EntryType Information -Message "Application started successfully."

# Logging a warning event
Write-EventLog -LogName System -Source MyApp -EventID 200 -EntryType Warning -Message "Potential misconfiguration detected."

# Registering a new event source and then writing an error
New-EventLog -LogName Application -Source MyNewApp
Write-EventLog -LogName Application -Source MyNewApp -EventID 500 -EntryType Error -Message "Critical failure encountered."

Cheat Sheet 

Write-EventLog -LogName <LogName> -Source <Source>
                -EventID <ID> -EntryType <Type> -Message <"Message">
