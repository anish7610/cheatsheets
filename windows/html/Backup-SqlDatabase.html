Backup-SqlDatabase Command

Description

The Backup-SqlDatabase cmdlet in Windows is used to create a backup of a SQL Server database. It supports full, differential, and log backups.

Common Options

Option Description
-ServerInstance Specifies the SQL Server instance to connect to.
-Database Specifies the name of the database to back up.
-BackupFile Specifies the file path to store the backup.
-BackupAction Specifies the type of backup: Full, Differential, or Log.
-CompressionOption Controls whether or not the backup is compressed.

Usage Examples 

# Full backup of a database
Backup-SqlDatabase -ServerInstance "MyServer" -Database "MyDatabase" -BackupFile "C:\Backups\MyDatabase.bak" -BackupAction Database

# Differential backup of a database
Backup-SqlDatabase -ServerInstance "MyServer" -Database "MyDatabase" -BackupFile "C:\Backups\MyDatabase_Diff.bak" -BackupAction Differential

# Log backup of a database with compression
Backup-SqlDatabase -ServerInstance "MyServer" -Database "MyDatabase" -BackupFile "C:\Backups\MyDatabase_Log.trn" -BackupAction Log -CompressionOption On

Cheat Sheet 

Backup-SqlDatabase -ServerInstance "Server" -Database "DB" -BackupFile "Path" -BackupAction {Database|Differential|Log} [-CompressionOption {On|Off}]
