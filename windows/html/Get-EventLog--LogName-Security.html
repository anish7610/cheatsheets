Windows Command: Get-EventLog -LogName Security

Description

The Get-EventLog command retrieves entries from the Windows event logs. The -LogName Security option specifically targets the Security event log, which contains records related to security and auditing events.

Common Options

Option Description
-LogName Specifies the name of the event log to retrieve entries from.
-Newest Retrieves the most recent specified number of entries.
-EntryType Filters entries by type (Error, Warning, Information).
-InstanceId Specifies a particular event ID to find specific entries.
-Before Returns entries logged before a specific date and time.
-After Returns entries logged after a specific date and time.

Usage Examples 

# Retrieve the 10 most recent security events
Get-EventLog -LogName Security -Newest 10

# Get all security events with the event ID 4625 (failed logon attempts)
Get-EventLog -LogName Security -InstanceId 4625

# Find security events recorded after January 1, 2023
Get-EventLog -LogName Security -After "01/01/2023"

# Fetch security events of type 'Error'
Get-EventLog -LogName Security -EntryType Error

Cheat Sheet 

# Recent entries
Get-EventLog -LogName Security -Newest X

# Filter by event ID
Get-EventLog -LogName Security -InstanceId ID

# Date filtering
Get-EventLog -LogName Security -After "MM/DD/YYYY"
Get-EventLog -LogName Security -Before "MM/DD/YYYY"

# Filter by type
Get-EventLog -LogName Security -EntryType Type
