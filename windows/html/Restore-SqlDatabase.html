Restore-SqlDatabase Command

The Restore-SqlDatabase command is used in Windows to restore a SQL database backup from a backup file.

Common Options

Option Description
-ServerInstance Specifies the SQL Server instance to connect to.
-Database Names the database to restore.
-BackupFile Provides the path to the backup file.
-NoRecovery Leaves the database in a restoring state.
-ReplaceDatabase Overwrites an existing database of the same name.

Usage Examples 

# Restore a database from a backup file
Restore-SqlDatabase -ServerInstance "MYSERVER" -Database "MyDatabase" -BackupFile "C:\Backups\MyDatabase.bak"

# Restore a database and leave it in restoring state (useful for log shipping)
Restore-SqlDatabase -ServerInstance "MYSERVER" -Database "MyDatabase" -BackupFile "C:\Backups\MyDatabase.bak" -NoRecovery

# Restore and overwrite an existing database
Restore-SqlDatabase -ServerInstance "MYSERVER" -Database "MyDatabase" -BackupFile "C:\Backups\MyDatabase.bak" -ReplaceDatabase

Quick Reference Cheat Sheet 

-ServerInstance: SQL Server name
-Database: Target database name
-BackupFile: Path to backup
-NoRecovery: Restore state without recovery
-ReplaceDatabase: Overwrite existing db
