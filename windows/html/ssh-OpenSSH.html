SSH (OpenSSH)

Description

SSH (Secure Shell) is a protocol for securely connecting to remote systems over a network. It provides encrypted communications and various authentication methods.

Common Options

Option Description
-l login_name Specify the user to log in as on the remote machine.
-i identity_file Use the specified file as the private key for authentication.
-p port Connect to the specified port on the remote host.
-v Verbose mode. Increase verbosity for debugging.
-N Do not execute a remote command (useful for port forwarding).
-T Disable pseudo-terminal allocation.
-L local_port:remote_host:remote_port Specify local port forwarding.

Examples 

# Connect to a remote host
ssh user@example.com

# Use a specific private key
ssh -i ~/.ssh/id_rsa user@example.com

# Connect to a remote host on a non-standard port
ssh -p 2222 user@example.com

# Enable verbose mode for more details
ssh -v user@example.com

# Local port forwarding example
ssh -L 8080:localhost:80 user@example.com

# Connect without executing remote commands (useful for tunneling)
ssh -N -L 8080:localhost:80 user@example.com

Cheat Sheet 


# Basic SSH connection
ssh user@hostname

# Use a private key
ssh -i path_to_key user@hostname

# Connect on a custom port
ssh -p port user@hostname

# Verbose output
ssh -v user@hostname

# Local port forwarding
ssh -L local_port:remote_host:remote_port user@hostname
