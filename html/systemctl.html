Systemctl Command Overview

Description

The systemctl command is a tool to examine and control the systemd system and service manager. It allows users to manage system services, check statuses, and control units.

Common Options

Option Description
start [unit] Starts the specified service or unit.
stop [unit] Stops the specified service or unit.
restart [unit] Restarts the specified service or unit.
status [unit] Shows the status of the specified service or unit.
enable [unit] Enables the service or unit to start on boot.
disable [unit] Disables the service or unit, preventing it from starting on boot.
list-units Lists all loaded units.
daemon-reload Reloads the systemd manager configuration.

Usage Examples 


# Start the Apache service
systemctl start apache2

# Stop the Apache service
systemctl stop apache2

# Restart the Apache service
systemctl restart apache2

# Check the status of the Apache service
systemctl status apache2

# Enable the Apache service to start at boot
systemctl enable apache2

# Disable the Apache service from starting at boot
systemctl disable apache2

# List all active systemd units
systemctl list-units
