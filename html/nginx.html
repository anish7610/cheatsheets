Understanding the nginx Command

Description

nginx is a high-performance HTTP server and reverse proxy, as well as an IMAP/POP3 proxy server.

Common Options

Option Description
-t Test the configuration file for syntax errors.
-s signal Send a signal to the master process. Signals can include stop , quit , reload , or reopen .
-v Display the version of nginx.
-V Display the version and configure options.
-c file Use an alternate configuration file.
-p prefix Set the prefix path (default is /usr/local/nginx/ ).

Usage Examples 

# Start nginx with the default configuration
nginx

# Test the current configuration file for syntax errors
nginx -t

# Reload nginx to apply changes in the configuration
nginx -s reload

# Stop the nginx server
nginx -s stop

# Start nginx with a specific configuration file
nginx -c /path/to/config/file.conf
