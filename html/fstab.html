Understanding the Linux fstab File

Description

The fstab file is a system configuration file in Linux that contains information about disk drives and partitions. It is used to define how disk drives should be automatically mounted at boot time.

Most Common Options

Option Explanation
defaults Uses the default mount options: rw, suid, dev, exec, auto, nouser, and async.
noauto Does not mount the filesystem at boot.
ro Mounts the filesystem as read-only.
rw Mounts the filesystem as read-write.
user Allows any user to mount and unmount the filesystem.
nouser Allows only root to mount and unmount the filesystem.
exec Allows execution of binaries.
noexec Prevents execution of binaries.

Usage Examples 

# Mount a USB drive to /mnt/usb
UUID=1234-5678 /mnt/usb vfat defaults,noauto,user 0 0

# Mount a read-only backup partition
/dev/sdb1 /backup ext4 defaults,ro 0 0

# NFS share mount
server:/export /mnt/nfs nfs defaults 0 0

Cheat Sheet 

# fstab entry format
<file system> <mount point> <type> <options> <dump> <pass>

# Example entry
/dev/sda1 / ext4 defaults 0 1
