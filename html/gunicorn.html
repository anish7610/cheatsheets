Gunicorn Command Overview

Description

Gunicorn, short for "Green Unicorn," is a Python WSGI HTTP Server for UNIX. It is a pre-fork worker model, which means it creates multiple worker processes to handle requests, allowing for efficient load handling in Python web applications.

Common Options

Option Description
-w Number of worker processes for handling requests.
-b Bind to a specific HOST:PORT.
-t Worker timeout in seconds.
--log-level Defines logging level (e.g., debug, info, warning).
--access-logfile Specifies access log file path.
--daemon Run Gunicorn as a background process.

Usage Examples 

# Run a simple application with default settings
gunicorn myapp:app

# Specify number of workers and binding address
gunicorn -w 4 -b 127.0.0.1:8000 myapp:app

# Set a timeout and logging level
gunicorn -t 120 --log-level=info myapp:app

# Run as a daemon with access log
gunicorn --daemon --access-logfile /var/log/gunicorn/access.log myapp:app

Cheat Sheet 

# Basic run
gunicorn myapp:app

# With options
gunicorn -w  -b : myapp:app
