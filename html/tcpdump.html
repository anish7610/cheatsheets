Linux tcpdump Command

Description

The tcpdump command is a powerful network packet analyzer used to capture and display the traffic passing through a network interface.

Common Options

Option Description
-i [interface] Specify network interface to listen on (e.g., eth0).
-n Don't convert addresses to names.
-nn Don't convert protocol and port numbers to names.
-w [file] Write the captured packets to a file for later analysis.
-r [file] Read packets from a file, instead of capturing live data.
-c [count] Stop capturing after receiving the specified number of packets.
-v Provide verbose output.
-vv Provide even more verbose output.
-X Show packet contents in both hex and ASCII.

Usage Examples 

# Capture packets on interface eth0
tcpdump -i eth0

# Capture 10 packets on eth0 without resolving hostnames
tcpdump -i eth0 -c 10 -n

# Write captured packets to a file
tcpdump -i eth0 -w capture.pcap

# Read packets from a file
tcpdump -r capture.pcap

# Capture TCP traffic on the default interface
tcpdump tcp

# Capture packets to a specific host
tcpdump host 192.168.1.1

# Capture packets on port 80
tcpdump port 80

Cheat Sheet
