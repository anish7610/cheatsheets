Linux Command: dig

dig (Domain Information Groper) is a flexible command-line tool used for querying DNS name servers to retrieve information about host addresses, mail exchanges, name servers, and related DNS records.

Common Options

Option Description
-t [type] Specify the type of DNS record to query (e.g., A , MX , NS , TXT ).
+short Provide a concise output, displaying just the answer to the query.
+nocmd Suppress the initial query output (useful for cleaner outputs).
+noall +answer Display only the answer section in the output.
@server Specify a DNS server to query instead of using the default resolver.
-x [address] Perform a reverse DNS lookup on an IP address.

Example Usage 

# Query the A record for a domain
dig example.com

# Query the MX (Mail Exchange) record for a domain
dig -t MX example.com

# Get a concise output of the A record
dig +short example.com

# Query a specific DNS server
dig @8.8.8.8 example.com

# Perform a reverse DNS lookup
dig -x 93.184.216.34

# Query and display only the answer section
dig +noall +answer example.com

Cheat Sheet 


dig example.com              # Default query
dig -t MX example.com        # Query MX records
dig +short example.com       # Short answer
dig @8.8.8.8 example.com     # Use specific DNS server
