Linux Command: mongod

Description

The mongod command is the primary daemon process for MongoDB. It handles data requests, manages data access, and performs background management operations.

Common Options

Option Description
--dbpath <path> Specify the directory where MongoDB stores data.
--port <number> Set the port on which MongoDB listens for connections (default is 27017).
--logpath <file> Specify the file where the log output is written.
--fork Run mongod as a background process.
--auth Enable authentication for connecting to the database.
--bind_ip <IP> Set the IP address that MongoDB binds to (default is localhost).

Usage Examples 

# Start MongoDB with default settings
mongod

# Start MongoDB with a specific data directory
mongod --dbpath /data/db

# Start MongoDB on a different port
mongod --port 28017

# Start MongoDB and write logs to a specific file
mongod --logpath /var/log/mongodb/mongod.log --fork

# Start MongoDB with authentication enabled
mongod --auth

# Bind MongoDB to a specific IP address
mongod --bind_ip 127.0.0.1

Cheat Sheet 

mongod --dbpath <path> --port <number> --logpath <file> --fork --auth --bind_ip <IP>
